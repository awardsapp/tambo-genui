The Docker images can be deployed to Kubernetes. Below is a minimal example covering all three services. For production, add resource limits, health checks, and ingress configuration appropriate for your cluster.

Unlike Docker Compose, Kubernetes does not guarantee startup order. The API and web pods may restart a few times before PostgreSQL is ready — this is normal, and Kubernetes keeps retrying until connections succeed.

> **Want Helm charts or a more detailed Kubernetes setup?** [File an issue](https://github.com/genui-ai/genui/issues) or reach out on [Discord](https://discord.gg/dJNvPEHth6) and we'll prioritize it.

## Secrets

Create a Kubernetes secret with your environment variables:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: genui-secrets
type: Opaque
stringData:
  POSTGRES_USER: postgres
  POSTGRES_PASSWORD: your-secure-password
  POSTGRES_DB: genui
  DATABASE_URL: postgresql://postgres:your-secure-password@genui-postgres:5432/genui
  API_KEY_SECRET: your-32-character-api-key-secret
  PROVIDER_KEY_SECRET: your-32-character-provider-secret
  NEXTAUTH_SECRET: your-nextauth-secret
  NEXTAUTH_URL: https://your-domain.com
  FALLBACK_OPENAI_API_KEY: your-openai-api-key
```

## PostgreSQL

For production, consider a managed database (e.g., AWS RDS, GCP Cloud SQL). If running PostgreSQL in-cluster:

```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: genui-postgres
spec:
  serviceName: genui-postgres
  replicas: 1
  selector:
    matchLabels:
      app: genui-postgres
  template:
    metadata:
      labels:
        app: genui-postgres
    spec:
      containers:
        - name: postgres
          image: postgres:17
          ports:
            - containerPort: 5432
          envFrom:
            - secretRef:
                name: genui-secrets
          volumeMounts:
            - name: postgres-data
              mountPath: /var/lib/postgresql/data
  volumeClaimTemplates:
    - metadata:
        name: postgres-data
      spec:
        accessModes: ["ReadWriteOnce"]
        resources:
          requests:
            storage: 10Gi
---
apiVersion: v1
kind: Service
metadata:
  name: genui-postgres
spec:
  selector:
    app: genui-postgres
  ports:
    - port: 5432
      targetPort: 5432
```

## API service (NestJS)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: genui-api
spec:
  replicas: 2
  selector:
    matchLabels:
      app: genui-api
  template:
    metadata:
      labels:
        app: genui-api
    spec:
      containers:
        - name: api
          image: your-registry/genui-api:latest
          ports:
            - containerPort: 3000
          env:
            - name: PORT
              value: "3000"
          envFrom:
            - secretRef:
                name: genui-secrets
---
apiVersion: v1
kind: Service
metadata:
  name: genui-api
spec:
  selector:
    app: genui-api
  ports:
    - port: 8261
      targetPort: 3000
```

## Web service (Next.js)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: genui-web
spec:
  replicas: 2
  selector:
    matchLabels:
      app: genui-web
  template:
    metadata:
      labels:
        app: genui-web
    spec:
      containers:
        - name: web
          image: your-registry/genui-web:latest
          ports:
            - containerPort: 3000
          env:
            - name: PORT
              value: "3000"
            # NEXT_PUBLIC_GENUI_API_URL must be reachable from the browser,
            # not an internal cluster address. Set this to your external API URL.
            - name: NEXT_PUBLIC_GENUI_API_URL
              value: https://api.your-domain.com
          envFrom:
            - secretRef:
                name: genui-secrets
---
apiVersion: v1
kind: Service
metadata:
  name: genui-web
spec:
  selector:
    app: genui-web
  ports:
    - port: 8260
      targetPort: 3000
```

## Database migrations

After deploying, run database migrations by executing into a running API pod:

```bash
kubectl exec -it deployment/genui-api -- npm -w @workspace-cloud/db run db:migrate
```

## Build and push images

```bash
./scripts/cloud/genui-build.sh
```

Tag and push to your container registry:

```bash
docker tag genui-web:latest your-registry/genui-web:latest
docker tag genui-api:latest your-registry/genui-api:latest
docker push your-registry/genui-web:latest
docker push your-registry/genui-api:latest
```
