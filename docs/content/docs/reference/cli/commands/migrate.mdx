`npx genui migrate`

Migrates Genui components from the legacy `components/ui` directory to the new `components/genui` directory and updates import paths automatically.

This command addresses the change in Genui CLI's installation approach - earlier versions installed components in `components/ui`, while current versions use the dedicated `components/genui` directory for better organization.

**What it does:**

1. Moves component files from `ui/` to `genui/` directory
2. Updates import paths in moved files
3. Preserves custom (non-Genui) components in original location
4. Shows preview of changes before applying

**Examples:**

```bash
# Preview migration changes (recommended first)
npx genui migrate --dry-run

# Perform migration with confirmation
npx genui migrate

# Skip confirmation prompts
npx genui migrate --yes
```

**Manual Migration Steps:**

If you prefer to migrate manually:

1. **Move Files**: Move all `.tsx` files from `/components/ui/` to `/components/genui/`
2. **Update Imports**: Change all imports from `@/components/ui/` to `@/components/genui/`
3. **Update Registry**: Update component registrations in `lib/genui.ts`
4. **Test**: Verify all components still work correctly
